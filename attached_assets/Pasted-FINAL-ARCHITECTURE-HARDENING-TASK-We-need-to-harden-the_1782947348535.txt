1. SQLite Write Queue Hardening (CRITICAL)

Current issue: write buffer may grow unbounded under burst traffic.

Implement:

* Add bounded queue size for SQLite writes
* If queue exceeds threshold:
    * coalesce duplicate writes by key (e.g. FID)
    * drop redundant intermediate updates
* Ensure flush runs every 100ms OR earlier if threshold reached
* Guarantee single active flush transaction at a time

Expected outcome:

* no memory explosion under 10K write bursts
* stable event loop latency

⸻

2. SWR + SingleFlight Global Lock Fix (CRITICAL)

Ensure cache refresh does NOT cause stampede.

Implement:

* Global request coalescing per key (FID / feed / profile)
* Only ONE active refresh per key at any time across all users
* All concurrent requests must:
    * receive stale data immediately
    * await shared in-flight promise if refresh is ongoing

Add strict guarantee:

* no duplicate Neynar calls for same key during refresh window
