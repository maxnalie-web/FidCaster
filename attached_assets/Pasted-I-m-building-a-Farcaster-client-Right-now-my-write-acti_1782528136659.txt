I'm building a Farcaster client. Right now my write actions (like, follow, cast) are broken because I'm using Neynar API with a shared API key which doesn't scale per user.

I need to migrate to a fully self-hosted flow where:
- Each user provides their Farcaster mnemonic once during onboarding
- We derive their Ed25519 signer from it and register it onchain on Optimism via KeyGateway contract
- We store the signer private key encrypted in the database (never store the mnemonic)
- All write actions (like, follow, recast, cast, unfollow) sign messages locally and submit directly to Farcaster hub — no Neynar API needed

   - returns { signerPrivateKey: string (hex), signerPublicKey: string (hex), txHash: string }

   - returns a NobleEd25519Signer instance from @farcaster/hub-nodejs

Use this EIP-712 domain for signing:
{
  name: 'Farcaster SignedKeyRequestValidator',
  version: '1',
  chainId: 10,
  verifyingContract: '0x00000000fc700472606ed4fa22623acf62c60553'

SignedKeyRequest: [
  { name: 'requestFid', type: 'uint256' },
  { name: 'key', type: 'bytes' },
  { name: 'deadline', type: 'uint256' },
